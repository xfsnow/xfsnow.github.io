# ChatGG - AI数学绘图助手

## 概述
ChatGG 是一个通过 AI 对话生成 GeoGebra 图形的简洁版程序。用户可以用自然语言描述数学作图需求，AI 会自动生成相应的 GeoGebra XML 命令并实时渲染绘图结果。

## 功能特性
- 🤖 **AI 智能绘图**：通过自然语言描述生成 GeoGebra 几何图形
- 💬 **流式对话**：支持实时流式响应，边生成边显示
- 🎨 **画板内嵌渲染**：自动解析 AI 返回的 XML 命令，实时在对话中渲染 GeoGebra 画板
- 🖼️ **多模态支持**：支持图片上传，可结合图片提问
- 📋 **一键复制命令**：每条绘图回复都附带可复制的 GeoGebra XML
- ⚙️ **多模型支持**：兼容任何 OpenAI 格式的 API（OpenAI、智谱、Azure 等），可自由配置端点、密钥与模型
- 💾 **对话历史**：自动保存本地对话记录，支持切换、删除与恢复
- 🪟 **侧边栏**：左侧历史记录与设置入口，支持自动折叠
- 📱 **响应式设计**：适配各种屏幕尺寸，移动端友好
- 💾 **本地存储**：自动保存 API 设置与对话历史到 localStorage
- 📐 **XML 格式命令**：使用 GeoGebra 官方 XML 格式，确保命令正确执行

## 技术架构

### 核心组件

#### 1. AI 客户端 ([ai.js](file:///d:/Study/ggb/chatgg/ai.js))
- 实现 `AiClient` 类，支持 OpenAI 兼容的流式 API
- 处理 SSE（Server-Sent Events）流式响应
- 通过回调 `onUpdate` / `onComplete` / `onError` 向主控制器推送数据
- 自动拼接增量内容，容错处理 JSON 解析错误

#### 2. 主控制器 ([chatxml.js](file:///d:/Study/ggb/chatgg/chatxml.js))
- 管理对话历史（`messageHistory`）和消息渲染
- 处理用户输入、图片附件与发送逻辑
- 集成设置面板、localStorage 持久化、历史记录管理
- 格式化 AI 回复内容：支持标题、加粗、表格、代码块等 Markdown 语法
- 自动提取 `ggb-xml` 代码块，并触发画板渲染
- 维护 GeoGebra 画板创建队列，避免并发冲突
- 通过 `ResizeObserver` 自适应调整画板尺寸
- 执行 XML 命令使用 `ggbApi.evalXML()` 方法

#### 3. 用户界面 ([index.html](file:///d:/Study/ggb/chatgg/index.html) + [chatgg.css](file:///d:/Study/ggb/chatgg/chatgg.css))
- **左侧边栏**：新对话、历史记录列表、设置入口
- **右侧主区域**：聊天头部、消息列表、输入框
- **设置弹窗**：配置 API 端点、API 密钥、模型名称
- **图片预览**：上传后可预览，点击 × 移除

### GeoGebra 画板渲染流程
1. AI 返回包含 ` ```ggb-xml ``` ` 代码块的回复
2. 前端 `extractGGBCommandsFromText()` 解析 XML 内容
3. `createGGBApplet()` 将创建请求加入队列，顺序执行避免冲突
4. `ensureGeoGebraLoaded()` 等待 GeoGebra 库加载完成（首次自动加载）
5. 注入 Applet，使用 `evalXML()` 执行 XML 命令
6. 依据 viewRange 与容器比例自动调整坐标系统（保持 1:1 比例）
7. XML 代码以独立区域渲染，支持一键复制

## 使用方法

### 1. 首次使用配置
1. 点击左下角的「设置」按钮
2. 输入接入端点（默认：`https://api.openai.com/v1`）
3. 输入 API 密钥
4. 输入模型名称（默认：`gpt-4o`）
5. 点击「保存设置」

> 配置保存到浏览器 `localStorage`，下次访问自动加载。

### 2. 开始对话
在输入框中输入数学作图需求，例如：
- "画一个等边三角形"
- "构造圆的切线"
- "证明勾股定理"

也可以点击欢迎页面的示例标签快速开始，或点击 📎 按钮上传图片辅助提问。

### 3. 对话管理
- 点击「新对话」按钮清空当前对话（自动保存）
- 点击侧边栏切换按钮隐藏/显示侧边栏
- 点击历史记录项可恢复过往对话
- 点击历史记录项的 × 按钮可删除对话
- 按 Enter 发送消息，Shift+Enter 换行

## 系统提示词
AI 被配置为 GeoGebra 几何绘图专家，遵循以下规则：

### XML 格式铁律
1. **对象标识**：使用 `label="..."` 而不是 `name="..."`
2. **点定义**：必须使用齐次坐标，平面点包含 `z="1.0"`
3. **公式表达式**：使用 `<expression>` 标签，公式写在 `exp="..."` 属性里，用方括号 `[]`
4. **几何构造**：使用 `<command name="命令">` 块，参数用 `a0, a1, a2...`

### 构造规则
- **固定点/数值**：`<element type="point" label="名称"><coords x="0.0" y="0.0" z="1.0"/></element>`
- **公式计算**：`<expression label="名称" exp="公式" type="类型"/>`
- **几何作图**：`<command name="命令"><input a0="A" a1="B"/><output a0="结果"/></command>`

### 输出格式
1. 先用自然语言解释绘图思路和步骤
2. 将 GeoGebra 命令放在 ` ```ggb-xml ``` ` 代码块中
3. 最后补充说明或注意事项

**示例输出：**
```
我来画一个等边三角形。

首先定义点A和B，然后通过旋转得到点C，最后用Polygon命令连接成三角形。

ggb-xml
<construction>
  <element type="point" label="A">
    <coords x="0.0" y="0.0" z="1.0"/>
  </element>
  <element type="point" label="B">
    <coords x="4.0" y="0.0" z="1.0"/>
  </element>
  <expression label="C" exp="Rotate[B, 60°, A]" type="point"/>
  <command name="Polygon">
    <input a0="A" a1="B" a2="C"/>
    <output a0="triangle"/>
  </command>
  <element type="polygon" label="triangle">
    <show object="true" label="false"/>
    <objColor r="0" g="100" b="0" alpha="0.1"/>
  </element>
</construction>


这样就完成了等边三角形的绘制。
```

**优势：**
- ✅ 使用官方 XML 格式，确保命令正确执行
- ✅ 支持复杂几何构造
- ✅ 可以在对话中直接渲染 GeoGebra 画板
- ✅ 支持多个画板同时显示
- ✅ 提供命令复制按钮

- Chrome / Edge（推荐）
- Firefox
- Safari
- 需要支持 ES6+、Fetch API 与 `ResizeObserver` 的现代浏览器

## 注意事项
1. 使用前必须配置有效的 API 密钥
2. API 调用会产生费用，请注意用量
3. GeoGebra 库从 CDN 加载，需要网络连接
4. 首次进入页面时会预加载 GeoGebra 库，最多等待 20 秒
5. 图片作为多模态消息随对话一起发送，可能消耗更多 Token
6. GeoGebra XML 命令必须严格遵循官方格式，否则会静默失败

## 响应速度优化方案

针对一次性提问AI响应时间较长的问题，以下是进一步优化方向：

### 方案1：题库解答预加载

**目标**：节省AI解题时间，专注于画图命令生成

**实现思路**：
- 从现有题库中提取题目对应的解答步骤
- 将题目描述和解答步骤一起作为输入传递给AI
- AI只需根据解答步骤生成绘图命令，无需自行解题

**预期效果**：减少AI"思考"时间，专注于绘图逻辑

---

### 方案2：分步交互式绘图

**目标**：提升用户体验，避免长时间等待

**实现思路**：
- 按照几何图形的逻辑关系，分步生成画图命令
- 每完成一步后展示当前图形状态
- 等待用户确认后再继续下一步
- 支持用户在任何步骤进行调整或回退

**预期效果**：
- 用户可以实时看到绘图进度
- 及时发现问题并调整
- 减少一次性等待时间

---

### 方案3：图片对比自动优化

**目标**：通过图片对比，自动优化画图命令质量

**实现思路**：
- 使用Python脚本与AI进行多轮对话
- AI生成画图命令后，执行并截图
- 将生成的截图与题目提供的静态图片进行对比
- 根据对比结果，让AI调整命令参数
- 重复迭代直到图形相似度达到阈值

**预期效果**：
- 自动优化画图准确性
- 减少人工干预
- 适合批量处理题库
