## 扩展测试：Functions + Angle + If + Slider（4 类型代表页）

- **测试 4 类页面**：Functions（topic）、Angle（command/长）、If（command/条件语法）、Slider（tool）
- **发现致命问题：`<em>` 标签一刀切跳过导致大量英文漏翻**
  - `<em>` 在 GeoGebra 文档中既包裹数学变量（x, y, 45°）又包裹英文单词（radians, Name, Interval, Then, true, false）
  - 之前将 `<em>` 整体加入 SKIP_TAGS 是错误的：Angle 页 6 处 "radians" 未翻译、If 页 "Then/true/false" 全保留英文、Slider 页 "Name/Interval/Speed/Animation" 全未翻译
- **修复方案**：从 SKIP_TAGS 移除 `<em>`，改为智能判断——只跳过纯数学变量（单字符或数字+符号组合，如 x, 45°, 90°, 1.57），翻译其他 `<em>` 内容
- **新增自动替换**：
  - explore 面板 `<span class="version">` 语言标签 English/英语 → 中文
  - CSS `content: "Note"` → `content: "注意"`
- **术语表新增**：radians→弧度, degrees→度数, true→真, false→假, undefined→未定义, must→必须, Direction→方向, restricted to interval→限制在区间, Derivative of→的导数, Slider Tool→滑动条工具, Note→注意（共 12 条，总计 222 条）
- **移除歧义术语**：Then 和 Else 从术语表移除（作为命令参数标签时不应翻译为 "则"/"否则"）
- **已知限制：文本碎片化问题**
  - `<p>` 中 `<em>`/`<code>`/`<a>` 标签将句子切碎为独立文本节点，AI 翻译时丢失上下文
  - 主要影响 If 命令页（大量 `<em>true</em>` `<em>false</em>` 标签）和 Slider 工具页（`<a>` 链接切断句子）
  - 根本解决需要重构为"整段翻译→回填"架构，当前作为已知限制接受
- **重新翻译统计**（修复后）：Angle 术语 67/166（提升自 52/149），If 64/162（提升自 54/148），Slider 58/131（提升自 53/113）
